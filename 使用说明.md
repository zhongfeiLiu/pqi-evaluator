## 第二步：在 Notion 中添加 Webhook 按钮

这是让系统自动运行的关键步骤。

### 方法 A：在数据库视图中添加按钮列（推荐）

1. 打开 **PQI 导师评估数据库**
2. 点击最右侧的 **"+"** 添加新属性
3. 选择属性类型为 **"Button"**（按钮）
4. 按钮名称填写：`🚀 开始评估`
5. 点击按钮的 **"Edit automation"**（编辑自动化）
6. 点击 **"Add action"** → 选择 **"Send Webhook"**
7. 填写以下信息：
   - **URL**: `https://web-production-a5794.up.railway.app/evaluate`
   - **Method**: `POST`
   - **Headers**: 添加一行
     - Key: `X-Webhook-Secret`
     - Value: `pqi_webhook_secret_2026`
   - **Body**: 选择 `JSON`，填写：
     ```json
     {
       "mentor_name": "{{prop('导师姓名')}}",
       "institution": "{{prop('学校/机构')}}",
       "notion_page_id": "{{id()}}"
     }
     ```
8. 点击 **"Save"** 保存

### 方法 B：在页面中添加独立按钮

1. 在 Notion 页面中输入 `/button` 选择 **Button** 块
2. 按钮文字填写：`🚀 评估导师`
3. 点击 **"Add action"** → **"Send Webhook"**
4. 配置同上（但 Body 中需手动填写导师姓名和学校）

```
1. 在数据库中新建一行
2. 填写「导师姓名」（如：张伟）
```

> **注意**：评估过程是异步的，点击按钮后服务会立即返回"评估已启动"的响应，实际评估结果会在 30-60 秒内写回 Notion。

## PQI 3.0 评分说明

### 核心公式

```
PQI = 0.35×S + 0.20×P + 0.20×C + 0.15×L + 0.10×F
PQI_final = PQI - 0.5×D
```

### 各维度说明

| 维度 | 权重 | 评估内容 |
|------|------|----------|
| **S** 学术产出质量 | 35% | 学生论文质量、期刊分区、第一作者发表情况 |
| **P** 研究生产力 | 20% | 导师近5年论文数量、h-index、引用量 |
| **C** 资源与支持 | 20% | 毕业生去向质量（海外博后、高校教职、大厂等） |
| **L** 实验室文化 | 15% | 实验室稳定性、建组年限、团队规模 |
| **F** 资金与稳定性 | 10% | 科研经费、基金项目情况 |
| **D** 惩罚项 | -0.5× | 退学率、延毕率（负向惩罚） |

### 评级标准

| 评级 | 分数范围 | 建议 |
|------|----------|------|
| **S 级** | ≥ 85 | 强烈推荐，顶尖导师 |
| **A 级** | 70-84 | 推荐，优秀导师 |
| **B 级** | 55-69 | 可考虑，中等水平 |
| **C 级** | 40-54 | 谨慎考虑，存在明显不足 |
| **D 级** | < 40 | 不推荐，风险较高 |

## 常见问题

**Q: 评估结果没有写回 Notion 怎么办？**

A: 可能原因：
1. Notion Integration 没有正确连接到数据库所在页面
2. 数据库 ID 配置有误

解决方法：确保在数据库所在的父页面中，通过 "..." → "Connections" → 添加 "PQI Evaluator" Integration。

**Q: 点击按钮后很久没有反应？**

A: 评估需要 30-60 秒，请耐心等待。如果超过 2 分钟仍无结果，可能是 Railway 服务休眠（免费版会在不活跃后休眠），再次点击按钮即可唤醒。

**Q: 评估结果准确吗？**

A: AI 基于 Google Scholar、学校官网、基金委等公开资料进行评估。评估报告中会注明每项数据的来源和置信度。由于部分数据（如退学率）难以从公开渠道获取，AI 会给出估算值并标注。**建议结合学长学姐反馈综合判断。**

**Q: Railway 免费版有什么限制？**

A: Railway 免费版每月提供 $5 额度（约 500 小时运行时间），对于个人使用完全够用。服务不活跃时会自动休眠，首次请求会有约 10-30 秒的冷启动时间。
